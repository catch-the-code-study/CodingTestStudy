## 🔸 참여 방법

### 1. 문제 신청 (✈️ 문제 등록하기)
* **Issues** 탭에서 풀고 싶은 문제를 등록해 주세요.

### 2. 소스 코드 업로드 및 리뷰 프로세스
1.  `Main` 브랜치에서 본인의 새 브랜치를 생성한다. (`Branch Naming 컨벤션` 확인)
2.  해결한 소스 코드를 본인 브랜치에 `push`한다.
3.  **Pull Request(PR)**를 통해 코드 리뷰를 요청한다.
4.  스터디원 피드백 반영 후 수정이 완료되면 **Label**을 변경한다.

<br/>

## 💬 Code Review 규칙

> **"리뷰는 숙제 검사가 아닌, 함께 성장하는 학습 과정입니다."**

* **칭찬은 아낌없이!** 👍 잘한 부분은 칭찬하고, 피드백이 없다면 따뜻한 한마디를 남겨주세요.
* **다른 시각 공유:** 전체 코드보다는 핵심 키워드나 간단한 접근 방식을 공유해 주세요.
* **충분한 설명:** 개선이 필요하다면 '왜' 그렇게 생각하는지 논리적으로 설명해 주세요.
* **스스로의 고민 유도:** 정답을 주기보다 스스로 개선 방법을 선택할 수 있는 질문을 던져주세요.
* **오픈 커뮤니케이션:** `~하는 건 어떨까요?`, `제안합니다` 등 부드러운 말투를 지향합니다.
* **비난 금지:** 사람에 대한 비판이 아닌, **코드 자체**에 집중하여 리뷰합니다.
